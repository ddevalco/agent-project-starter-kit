# Execution Plan (ExecPlan)

Use this file for complex work that spans multiple steps or touches many files.
Treat it as a living plan that guides execution and captures decisions.

## Context and Orientation

Describe the current state as if the reader knows nothing about the repo. List
relevant files and what they do. Define any non-obvious terms.

## Goal

State the user-visible outcome and success criteria.

## Plan of Work

Describe the sequence of changes in prose. Name files and what will change.

## Milestones

For each milestone, explain what will exist at the end and how to verify it.

## Concrete Steps

List the exact commands to run and where. Include expected outputs.

## Validation and Acceptance

How to prove the change works (tests, manual steps, outputs).

## Idempotence and Recovery

How to re-run steps safely or roll back if something fails.

## Decision Log

Record key decisions with short rationale and date.
